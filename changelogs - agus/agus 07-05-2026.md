## 6. Corrección de errores TypeScript (16 archivos)

Se corrigieron errores pre-existentes en los siguientes archivos:

| Error | Fix aplicado |
|---|---|
| TS18046 — `catch (e)` tipo unknown | `catch (e: any)` |
| TS7006 — parámetro sin tipo | `: any` en params de funciones y callbacks |
| TS7034/7005 — array/variable implícita | `let x: any[] = []`, `let cb: any = null` |
| TS7053 — indexación con string en `{}` | `Record<string, any>` o cast `(obj as any)[key]` |
| TS2564 — propiedad de clase sin inicializar | `= ''` como valor por defecto |
| TS2322 — `string \| undefined` | `apiKey!` (non-null assertion) |
| TS2345 — tipo incorrecto en argumento | `!!(expr)` para forzar `boolean` |
| TS18049 — posiblemente null/undefined | Optional chaining `?.` |

Archivos corregidos: `idleFlow`, `locationFlow`, `reconectionFlow`, `welcomeFlowDoc`, `queueManager`, `timeOut`, `supabaseAdapter`, `updateDoc`, `updateSheet`, `external_api.routes`, `AssistantBridge`, `openaiBridge`, `calendarEvents`, `metaDiscovery`, `ai.manager`, `AssistantResponseProcessor`.

## 9. Script `npm run dev:backoffice`

**Nuevo comando** que levanta el servidor y abre el backoffice automáticamente en el navegador cuando el server está listo.

```json
"dev:backoffice": "concurrently ... \"npm run dev\" \"node scripts/open-browser.js\""
```

**Nuevo archivo:** `scripts/open-browser.js`  
Hace polling a `/health` cada 2 segundos. Cuando el servidor responde 200, ejecuta `start` (Windows) / `open` (Mac) / `xdg-open` (Linux) para abrir `http://localhost:8080/backoffice`.

**Scripts disponibles:**

| Comando | Qué hace |
|---|---|
| `npm run dev` | Lint + nodemon (sin abrir browser) |
| `npm run dev:backoffice` | Lint + nodemon + abre el backoffice al estar listo |
| `npm run build` | Compila con esbuild a `dist/` |
| `npm start` | Corre el build compilado |
